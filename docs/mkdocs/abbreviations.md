# Skróty

Skróty pozwalają na definiowanie skrótów i akronimów, które są automatycznie rozwijane po najechaniu myszką. To przydatne do wyjaśniania terminów technicznych bez zakłócania przepływu tekstu.

## Użycie

Skróty definiuje się na początku dokumentu lub w dowolnym miejscu używając składni:

````markdown
*[HTML]: HyperText Markup Language
*[CSS]: Cascading Style Sheets
*[API]: Application Programming Interface
````

Następnie używasz skrótu w tekście normalnie:

````markdown
HTML jest językiem znaczników.
CSS służy do stylowania.
API pozwala na komunikację między aplikacjami.
````

*[URL]: Uniform Resource Locator

*[IDE]: Integrated Development Environment

*[GUI]: Graphical User Interface
*[SQL]: Structured Query Language
*[NoSQL]: Not Only SQL

*[MVC]: Model-View-Controller
*[REST]: Representational State Transfer
*[SOAP]: Simple Object Access Protocol
*[OAuth]: Open Authorization

### Podstawowe użycie

HTML jest językiem znaczników używany do tworzenia stron internetowych.

CSS służy do stylowania dokumentów HTML.

API pozwala na komunikację między różnymi aplikacjami.

### Skróty w kontekście

Aplikacja używa REST API do komunikacji z serwerem.

Dane są przesyłane w formacie JSON przez protokół HTTPS.

### Skróty w listach

- HTML - język znaczników
- CSS - język stylów
- JavaScript - język programowania
- API - interfejs programistyczny aplikacji

### Skróty w tabelach

| Skrót | Pełna nazwa |

### Skróty techniczne

Aplikacja używa SQL do zapytań do bazy danych.

IDE ułatwia programowanie dzięki różnym narzędziom.

CLI pozwala na zarządzanie aplikacją z linii poleceń.

### Skróty w admonitions

## Wskazówki

- Definiuj skróty na początku dokumentu lub w sekcji z definicjami
- Skróty są automatycznie rozwijane po najechaniu myszką
- Możesz używać tego samego skrótu wielokrotnie w dokumencie
- Skróty działają tylko w dokumencie, w którym są zdefiniowane
- Używaj skrótów do wyjaśniania terminów technicznych bez zakłócania przepływu tekstu
